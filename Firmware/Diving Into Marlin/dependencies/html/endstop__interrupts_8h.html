Without endstop interrupts the endstop pins must be polled continually in the stepper-ISR via endstops.update(), most of the time finding no change. With this feature endstops.update() is called only when we know that at least one endstop has changed state, saving valuable CPU cycles.

This feature only works when all used endstop pins can generate either an 'external interrupt' or a 'pin change interrupt'.

Test whether pins issue interrupts on your board by flashing 'pin_interrupt_test.ino'. (Located in Marlin/buildroot/share/pin_interrupt_test/pin_interrupt_test.ino) Patch for pins_arduino.h (....h)

These macros for the Arduino MEGA do not include the two connected pins on Port J (D13, D14). So we extend them here because these are the normal pins for Y_MIN and Y_MAX on RAMPS. There are more PCI-enabled processor pins on Port J, but they are not connected to Arduino MEGA.
